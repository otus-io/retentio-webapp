# 标签（Tags）

**标签（Tags）** 是你挂在 **卡组** 与 **事实** 上的标记，用来整理和筛选学习材料。标签属于 **你的账户**，不局限于某一个卡组——创建一次即可在多个卡组、多条事实上复用。

标签是可选的。没有标签也能正常学习与复习；当你需要分组（例如 `verb`、`JLPT N3`、`hard`）或只复习卡组中的一部分卡片时，标签会很有用。

## 标签与卡组、事实的关系

| 位置     | 作用                                                                     |
| -------- | ------------------------------------------------------------------------ |
| **卡组** | 标记整个卡组（如 `Japanese`、`exam-prep`）。每个卡组最多 **20** 个标签。 |
| **事实** | 标记卡组内的单条内容（如 `noun`、`food`）。一条事实可有多个标签。        |

**字段** 定义卡组的列结构；**标签** 是独立的元数据，用于筛选与归类。事实仍只保存 **条目**——标签与事实关联，不写在条目文本里。

同一名称的标签可以出现在多个卡组、多条事实上。删除标签会移除它在所有位置的关联。

## 创建与管理

添加标签有两种常见方式：

1. **在创建内容时按名称添加** — 创建卡组或批量添加事实时传入标签 **名称**（字符串）。若名称尚不存在，Rete 会自动创建并关联。
2. **单独管理标签库** — 创建、重命名或删除标签，再关联到卡组或事实。

每个标签有 **名称** 和可选的 **说明**。名称在规范化后须在你的账户内唯一（去首尾空格、合并连续空格、不区分大小写比较）。允许使用字母、数字、空格、连字符（`-`）和撇号（`'`）。

**上限：**

- 每位用户最多 **100** 个标签
- 每个卡组最多 **20** 个标签

## 给事实打标签

添加事实时，可为每条事实单独指定可选 **标签**——例如一条标 `verb`，另一条标 `food`。同一批中的不同事实可以有不同的标签，也可以不打标签。

事实创建后，仍可在不修改条目的前提下为单条事实增删标签。也可以只查询某条事实的标签列表，而无需加载完整事实内容。

若要查看该标签在你所有卡组中关联了哪些事实，可通过该标签的事实列表查询（API 中按 `tag_id`）。

## 给卡组打标签

创建卡组时同样可以传入可选 **标签**（名称），例如 `IELTS` 与 `vocabulary`。同一次请求中的重复名称会合并为一条关联。

之后也可以为已有卡组添加或移除标签。卡组标签便于在库中浏览与归类，不会改变单张卡片的正反面布局。

## 按标签学习

复习时，可以只学习某一卡组内、事实带有指定标签的卡片。获取下一张到期卡与卡片统计均支持按 `tag_id` 筛选，从而只纳入带该标签的事实所对应的卡片。

标签不改变卡片的正反面结构，只缩小 **哪些** 卡片参与选卡。

## 导入的卡组

从他人发布的卡组 **导入** 后，事实内容为只读，但 **标签归你所有**。你可以为导入的事实与卡组打自己的标签，用于个人筛选，而无需修改共享内容。

## 小结

标签是按用户维度的 **卡组** 与 **事实** 标记，用于整理与按标签复习。可在录入时用名称自动创建，也可单独维护；注意用户级与卡组级上限。标签与卡组 **字段**、事实 **条目** 互补——后者才承载实际学习内容。
