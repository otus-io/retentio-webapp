# 概览

为了更好地使用 Rete，建议你先了解几个核心概念。Rete 的内容组织、学习流程和复习机制，都是围绕这些基础概念展开的。

建议按以下顺序阅读本章节：

1. **概览**（本页）
2. [卡组](/guide/key-concepts/decks)
3. [事实](/guide/key-concepts/facts)
4. [卡片](/guide/key-concepts/cards)
5. [标签](/guide/key-concepts/tags)

以下说明 Rete 中的 **卡组**、**字段**、**事实**、**卡片** 与 **标签**。

## 整体关系

**卡组（Deck）** 是围绕某一主题或学习目标的学习容器：包含名称、列结构、每日新卡节奏，以及属于该卡组的所有事实与卡片。

**字段（Fields）** 是创建卡组时定义的列名（例如 `English` 与 `中文`），用来表示一条学习内容中每一列的含义。列名只保存在卡组上；添加 **事实** 时，你按相同顺序填写 **条目（entries）**——第 `0` 条对应第一列，第 `1` 条对应第二列，依此类推。

**事实（Fact）** 是底层的学习内容：一条完整的学习项（词对、概念、术语与释义等）。事实 **不等于** 卡片。事实保存原始材料；系统根据事实生成你实际复习的卡片。

**卡片（Card）** 是你每天复习的对象。每张卡片属于一个事实，有 **正面** 与 **背面**（由哪些条目显示在哪一侧决定），并带有间隔重复相关的调度信息（何时到期、是否隐藏等）。默认情况下，添加一条事实会生成 **一张** 卡片（正面 = 第一条目，背面 = 其余条目）。你也可以为同一事实增加 **兄弟卡片**——例如英译中与中译英——从而练习双向回忆。

文字、图片、音频与视频保存在事实的 **条目** 中（不是单独的「卡片文件」）。**字段** 只在卡组上表示列名；详见 [卡组](/guide/key-concepts/decks) 与 [事实](/guide/key-concepts/facts)。

## 一览

| 概念     | 作用                                                                            |
| -------- | ------------------------------------------------------------------------------- |
| **卡组** | 归纳某一主题的内容；定义 **字段**（列）与 **速率 rate**（每天引入多少张新卡）。 |
| **字段** | 卡组上的列名；事实按位置填写对应的 **条目**。                                   |
| **事实** | 你创建与编辑的内容（各列条目，可选媒体）。                                      |
| **卡片** | 针对某一事实的一种可复习的正反面布局，并有独立的学习进度。                      |

## 数量关系

- 一个卡组包含许多 **事实** 与许多 **卡片**。
- 一条事实通常先有 **一张** 卡片；需要多种方向或布局时，可有 **两张或更多** 卡片（兄弟卡）。
- 每张卡片在卡组内只对应 **一个** 事实。

## 典型流程

1. **创建卡组** — 设置字段（列）与每天 **新卡片** 数量（`rate`）。数值越大，新内容进入越快；越小，节奏越平缓。新卡会在一天内均匀分布。
2. **添加事实** — 为各列填写内容（及可选的图片、音频、视频）。系统为每条事实生成相应卡片。
3. **学习** — Rete 从卡组中选出当前最该复习的到期卡片；你先看到正面，再看到背面，评分会更新下次复习时间。
4. **维护** — 在你拥有的卡组上可编辑事实条目（从他人发布的卡组 **导入** 的副本中，事实内容为只读快照）；**删除事实** 会同时移除该事实下的所有卡片；**只删除某张卡片** 则保留事实及同一事实下的其他卡片。
