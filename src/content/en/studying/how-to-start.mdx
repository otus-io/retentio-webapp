# How to Start?

## Getting Started

In Retentio, users can start by creating a deck, adding facts, generating cards, and then studying and reviewing those cards. This process helps users gradually build their own memory training system.

For new users, the basic workflow includes registering an account, logging in, creating a deck, adding facts, generating cards, and starting the learning process. This allows users to organize learning content from scratch and turn scattered knowledge into cards that can be reviewed and remembered over time.

## Register and Log In

Users first need to create a Retentio account and log in. After registration, the user’s decks, facts, cards, and learning records will be saved under their own account.

After logging in, users can enter the main Retentio interface and begin managing their learning content. The account system ensures that users can continue accessing their previously created learning materials and review progress when they return later.

## Create a Deck

After logging in, users can go to the Decks tab. In the Decks tab, users can click the button in the upper-right corner to open the deck creation dialog.

In the dialog, users need to enter the basic information for the deck and define the fields used by that deck. Fields determine what types of information can be included in later facts, such as “English”, “中文”, “Example Sentence”, “Image”, or “Audio”.

After the deck is created, it will appear in the user’s deck library. Users can then enter the deck and continue adding learning content.

In the future, Retentio will also provide a selection of high-quality curated decks. Users will be able to import these decks directly into their own deck library. This means users do not always need to create all learning content from scratch, because apparently rebuilding everything manually is not the peak of civilization.

## Create Facts

After creating a deck, users need to create facts within that deck. Facts are the smallest learning units used to build cards and decks. A fact usually represents a piece of content that needs to be learned and remembered.

For example, in an English learning deck, users can create a fact like this:

`English: hello`  
`中文: 你好`

If the deck includes more fields, users can also add example sentences, images, audio, notes, or other related content.

Facts store the original learning content. They are not just cards shown directly to users. Instead, they provide the basic data used to generate cards later.

## Generate Cards

After facts are created, the system can generate corresponding cards based on the field content inside each fact.

In most cases, one fact can generate one card. For example:

Front: `hello`  
Back: `你好`

In some learning scenarios, one fact can also generate multiple cards. For the same fact, the system can generate both an English-to-Chinese card and a Chinese-to-English card:

Front: `hello`  
Back: `你好`

Or:

Front: `你好`  
Back: `hello`

This allows users to practice recall in different directions. Users can learn not only to see the English word and remember the Chinese meaning, but also to see the Chinese meaning and recall the English word.

## Start Learning

After the deck, facts, and cards are created, users can start learning.

During learning, users will see the front side of a card and try to recall the answer. Then they can check the back side of the card to confirm whether their memory was correct.

Through repeated learning and review, Retentio helps users strengthen their memory of the learning content. Decks are used to organize learning topics, facts are used to store learning content, and cards are used for the actual memory training process.

## Summary

The basic process of getting started with Retentio is: register and log in, go to the Decks tab to create a deck, add facts inside the deck, generate cards from those facts, and then start learning.

Users can create their own decks and learning content, or later import curated high-quality decks provided by Retentio. Through this process, users can quickly build their own learning library and begin a long-term, structured memory training workflow.
