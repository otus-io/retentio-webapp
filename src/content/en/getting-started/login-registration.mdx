# Login and Register

## Login and Register

Login and Register are the basic entry points to the Retentio account system. Through login, existing users can access their learning content, decks, facts, cards, and review records. Through registration, new users can create their own accounts and start using Retentio to manage their learning.

For an application like Retentio, which needs to store learning content and review progress over time, the account system is the foundation that connects each user with their personal learning data. After logging in or registering, users can manage their learning content more reliably and continue their learning process across different sessions.

## Login

Login is used to verify the identity of an existing user. The user needs to enter a username and password, and the system checks whether the account exists and whether the password is correct.

The username is used to identify the user account, while the password is used to protect account security. Only when the username and password match can the user enter Retentio and access the learning data stored under that account.

After logging in successfully, users can continue viewing their existing decks, studying cards, reviewing content, and checking their learning records. If the login information is incorrect, the system should prompt the user to check whether the username or password has been entered correctly.

## Register

Register is used to create a new Retentio account. New users need to fill in basic account information, including email, username, password, and confirm password. The system uses this information to create a new user account.

The email is usually used for account identification, notifications, or password recovery. The username is used to identify the user within the system. The password is used to protect the account and prevent unauthorized access to the user’s learning content.

After registration is complete, users can start creating their own learning content, such as creating decks, defining fields, adding facts, and learning through the generated cards.

## Password Confirmation

During registration, users need to enter both a password and a confirm password. The confirm password field is used to make sure the password has not been entered incorrectly.

The user needs to enter the same content in both password fields. This helps reduce the chance of login problems caused by typing mistakes and allows users to confirm their password setting more clearly when creating an account.

If the two passwords do not match, the system should prompt the user to check and correct them. Although this is a simple step, it helps improve the accuracy of the account creation process.

## Forgot Password

The login page also provides a forgot password entry. When users forget their password or cannot log in normally, they can use this entry to recover their password.

The forgot password feature helps users regain access to their accounts and avoid losing access to their existing learning content and review records.

For a long-term learning application, this feature is important because decks, facts, cards, and learning progress are all connected to the user account. If the account cannot be accessed, the user’s learning system becomes difficult to continue.

## Page Switching

The login and register pages usually provide entries for switching between each other.

If users do not have an account yet, they can go from the login page to the registration process. If users already have an account, they can return from the registration page to the login page.

This switching flow helps users in different situations quickly find the correct action path. Login, registration, and password recovery together form the basic account access process of Retentio.

## Summary

Login and Register are the core entry points of the Retentio account system. Login helps existing users access their learning content, while registration helps new users create an account and start using Retentio.

Through the account system, users’ decks, facts, cards, and review records can be saved under their personal accounts. This supports a long-term, continuous, and structured learning process. Login, registration, password confirmation, and password recovery together help users enter the Retentio learning system in a clear and secure way.
